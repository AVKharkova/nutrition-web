Комплексное решение для трекинга питания, состоящее из умного Telegram-бота и стильного веб-дашборда. Бот позволяет записывать еду текстом, голосом или по фото с помощью AI (OpenAI), а сайт предоставляет удобную аналитику, геймификацию (Острова Личности) и дашборд в эстетике **Neo Kinpaku**.

## 🌟 Возможности

### 🤖 Telegram-бот (n8n + AI)
- **Умный логгинг еды**: отправьте фото блюда, голосовое или текстовое сообщение — бот автоматически распознает еду, посчитает КБЖУ с помощью OpenAI.
- **Геймификация**: система "Острова Личности" и "Пульт Эмоций" (начисление прогресса за здоровое питание, соблюдение КБЖУ, активность).
- **Управление профилем**: расчет индивидуальной нормы калорий (формула Миффлина-Сан Жеора) в зависимости от пола, веса, роста и целей.
- **Трекинг воды**: быстрый логгинг выпитой воды через inline-кнопки.
- **Еженедельные дайджесты**: автоматические отчеты о прогрессе.

### 💻 Веб-дашборд (React / Vite)
- **Эстетика Neo Kinpaku**: премиальный темный лак, золото кинпаку (Kinpaku Gold) и патина (Verdigris Patina) (подробнее в `DESIGN.md`).
- **Авторизация**: вход через Telegram-виджет (бесшовная связка с ботом) или по email (Supabase Magic Links).
- **Разделы**:
  - **Сегодня**: текущий прогресс по калориям и макронутриентам.
  - **Дневник**: история приемов пищи.
  - **Острова**: визуализация геймификации и развития "Островов личности".
  - **Профиль**: управление настройками и целями.

## 🏗 Архитектура и стек

- **Бэкенд бота**: [n8n](https://n8n.io/) (разбит на модульные воркфлоу: Main Router, Food Logging, Callbacks, Profile, Gamification, Web Backend).
- **AI**: OpenAI API (распознавание текста, аудио и изображений).
- **База данных и Auth**: [Supabase](https://supabase.com/) (PostgreSQL + RLS политики).
- **Фронтенд**: React 18, Vite, кастомная CSS-система токенов.

## 🚀 Установка и настройка

### 1. Настройка Supabase
1. Создайте проект в Supabase.
2. Перейдите в SQL Editor и выполните скрипт из файла `supabase-setup.sql` (создание таблиц и RLS политик).
3. В разделе **Authentication → URL Configuration** добавьте адрес вашего сайта в *Site URL* и *Redirect URLs* (для локальной разработки: `http://localhost:5173`).
4. Скопируйте **Project URL**, **anon key** и **service_role key** (в Settings → API).

### 2. Настройка n8n (Бэкенд бота)
1. Установите и запустите n8n.
2. Импортируйте воркфлоу из JSON-файлов (`WF-0` ... `WF-8`). Важные:
   - `WF-0 Main Router.json` (точка входа)
   - `WF-8 Web Backend.json` (вебхуки для сайта)
3. Настройте Credentials в n8n (для Postgres и OpenAI).
4. Заполните переменные в узле **Config** (в `WF-8`): `supabase_url`, `service_key` (важно: использовать `service_role` ключ), `bot_token` (от @BotFather).
5. Активируйте воркфлоу и скопируйте Webhook URL для настройки связи с сайтом и ботом.
> **Note**: Если вход через Telegram возвращает ошибку `require('crypto')`, добавьте в окружение n8n переменную `NODE_FUNCTION_ALLOW_BUILTIN=crypto` (используется для HMAC подписи).

### 3. Telegram Login Widget
1. Перейдите в [@BotFather](https://t.me/BotFather).
2. Вызовите команду `/setdomain` для вашего бота (`netkann_nutritionbot`).
3. Укажите домен вашего сайта (виджет не работает с `localhost`, для локальной отладки используйте временный туннель типа ngrok/cloudflared или вход по почте).

### 4. Запуск Веб-дашборда
```bash
cd nutrition-web

# Скопируйте пример конфига и заполните значениями из шагов 1-2
cp .env.example .env

# Установите зависимости
npm install

# Запустите локальный сервер
npm run dev
```
Сайт будет доступен по адресу: `http://localhost:5173`

## 📦 Сборка для продакшена (Деплой)
```bash
npm run build
```
Папка `dist/` будет готова к деплою (Netlify, Vercel, Cloudflare Pages).
* После деплоя обязательно добавьте ваш рабочий домен в **Supabase** (Redirect URLs) и в **@BotFather** (`/setdomain`).

## 📝 Ограничения текущей версии (v1)
- Логгинг еды через сайт обновляет дневник, но механики геймификации (Острова, Эмоции) в данный момент начисляются только при работе через Telegram-бота (воркфлоу `WF-2`/`WF-6`).
- Регистрация и онбординг новых пользователей осуществляются **только через бота** (команда `/start`), так как онбординг интегрирован в логику n8n бота.
